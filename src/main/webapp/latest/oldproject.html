Projects zanata-server

This project currently contains strings for the Zanata website, but not for the web editor.

Notes for translators:

Source comments

Source comments have been added where there appears to be ambiguity

  • view these by hovering the cursor over the source string, or by expanding the string info panel.
  • Most strings can be found on the website, but some may only be visible to project maintainers or administrators.
  • Please let us know if a string is ambiguous or confusing and does not have adequate source commenting (email damason@redhat.com or file a bug with the link in the main menu).

String order

Strings that appear on every page (e.g. the menus and footer) are presented first, followed by strings for each individual page. Strings for some similar pages are grouped together.

Individual pages are ordered like in the menus above, with a 'depth-first' approach. e.g. Projects is first, then all the pages under it such as project view and version view, then Groups, then Languages and so on.

Where possible, strings are presented in the same order as they appear on the page (left-to-right and top-to-bottom). For example, on the current page "Documents:" is first (the "Version" column heading has already been included earlier), followed by the "Translate Links" column heading and some tooltips used in that column, followed by the "Manage Version" column heading, and so on.

Where are strings from?

The first string for a page is marked with source comments that show the "breadcrumb" address of the page, which matches the path shown next to the house icon on the line just below the project search. Navigate to this page on the website to see where strings are used in the user interface. A line of dashes is used around these to make them stand out from normal source comments.

e.g. ------ [home] > Projects > [project-id] ------

If multiple pages are grouped together, the breadcrumb address for each is shown on a separate line.

Variables in strings

Strings may contain xml tags, and variables in the form {var} and #{var}, which should not be translated. Some strings include other strings with a variable in the form #{messages['stringid']} where stringid is the Resource ID of the string that is included. Ideally these should only be used in translations when they appear in the source string.

View source files: https://github.com/zanata/zanata/blob/master/zanata-war/src/main/resources/messages.properties
